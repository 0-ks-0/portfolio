Residential Care Facility Management System - HemlockVillage

Optimizing Operations and Care with a Comprehensive Residential Facility Management System

Nov - Dec 2024

Laravel (PHP) HTML CSS JavaScript

Developed at Thaddeus Stevens College of Technology as a group final project for CSET220 - Software Project III.
A role-based web application designed to enhance operational efficiency in residential care facilities. The system aims to streamline key administrative tasks, improving both staff coordination and patient care.

Things I Learned

  • How to create my own PHP Artisan command.
  • How to create custom middleware for route authorization.
  • Defining relationships in models and using eager loading to improve query efficiency.
  • Understanding and implementing Laravel pagination.
  • First-time exposure to namespaces.

Challenges and Solutions

  • Running Migrations in the Correct Order: Laravel runs migrations in alphabetical order, which led to errors with foreign key constraints as tables may not exist at the time of running a migration. Solution: Created a custom PHP Artisan command to run migrations in a specific order.
  • Resetting Database: I needed a way to drop all the tables in the database to reset the database for reasons including testing purposes due to there being very minimal data at the time, and making it easy to reset if the tables or their structure changed. Because I created a custom php artisan command to run migrations in a custom order instead of renaming the files to force them to be in the correct order, I could not run `php artisan migrate:fresh` to easily do that. Solution: Created another custom Artisan command to delete all tables in the database.
  • Ensuring Consistent Data Across Group Members: To ensure everyone had the same data for testing, I used seeders to populate the database with predefined data, which can be done with a PHP Artisan command.
  • Avoiding Clutter with Imports: To reduce clutter in files, I restructured namespaces to minimize the number of imports of controllers and models needed.
  • Unauthorized Access: I created a middleware to handle both authentication and authorization for routes because I can easily group routes under this middleware.
  • Missing `kernel.php` Based on what I obtained from my research, Laravel 11 relocated the kernal.php file. However, the project did not have that file, so I could not register my created middleware. Solution: Made the middleware function static and manually call it in the route file.
  • Handling Large Datasets: To prevent performance issues with large datasets (e.g., the Users page), I implemented pagination to split records across multiple pages and improve load time. This allows users to navigate by the page rather than the position of it.
    I was not able to figure out how to map when using a LengthAwarePaginator--Laravel's object for pagination.

Things I Would Do Differently

  • Implement more pagination to enhance the user experience with large datasets.
  • Add date selection for pages to allow users to view historical data.
  • Enhance the functionality to allow more than four patient groups, allow more employees per roster, and allow patients to have multiple appointments per day, including keeping track of appointment times.
